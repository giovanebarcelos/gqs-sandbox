Pesquisa página do Google - Selenium

O objetivo deste tutorial será abrir a página do Google, realizar uma busca, e verificar se os resultados da busca aparecem.

Passos para Usar o Selenium IDE

1. Instalar o Selenium IDE
    Para Chrome:
        Vá para a Chrome Web Store.
        Pesquise por "Selenium IDE".
        Clique em "Adicionar ao Chrome" e depois "Adicionar extensão".

    Para Firefox:
        Vá para o site de complementos do Firefox.
        Pesquise por "Selenium IDE".
        Clique em "Adicionar ao Firefox" e depois "Adicionar".

2. Abrir o Selenium IDE
    Abra o navegador onde você instalou o Selenium IDE.
    Clique no ícone do Selenium IDE na barra de ferramentas do navegador para iniciar a aplicação.

3. Criar um Novo Projeto
    Quando o Selenium IDE abrir, clique em "Create a new project".
    Dê um nome ao seu projeto, por exemplo, "GoogleSearchTest", e clique em "OK".

4. Gravar um Teste
    Clique em "Record a new test in a new project".
    Digite a URL https://www.google.com e clique em "Start recording".
    O navegador abrirá a página do Google.

5. Realizar Ações no Site
    No Google, digite "Selenium WebDriver" na barra de pesquisa.
    Pressione "Enter" para realizar a busca.
    O Selenium IDE gravará todas essas ações.

6. Parar a Gravação
    Quando terminar a gravação, clique no botão "Stop recording" no Selenium IDE.
    Dê um nome ao seu teste, por exemplo, "SearchTest", e clique em "OK".

7. Editar o Teste Gravado
    No painel esquerdo do Selenium IDE, você verá uma lista de comandos gravados.
    Verifique se os comandos estão corretos e fazem sentido. Você pode editar comandos clicando duas vezes neles.
    Por exemplo, você deve ver comandos como open, type, e sendKeys.

8. Adicionar Verificação do Resultado
    Adicione um comando para verificar se os resultados da busca aparecem. Por exemplo, verifique se o elemento que contém o texto "Selenium WebDriver" está presente.
    Para fazer isso, adicione um comando assertText ou assertElementPresent. No Selenium IDE, isso pode ser feito manualmente:
        Clique em "Add command".
        Escolha assertText ou assertElementPresent no campo "Command".
        No campo "Target", coloque o seletor do elemento que você quer verificar (por exemplo, css=h3:contains('Selenium WebDriver')).

9. Executar o Teste
    Clique no botão "Run current test" (ícone de play) para executar o teste que você gravou.
    O Selenium IDE abrirá uma nova janela do navegador e executará as ações que você gravou.
    Você verá os resultados do teste no painel de resultados.

Exportar o Teste

O Selenium IDE permite exportar os testes para várias linguagens de programação e frameworks. Aqui está como exportar o teste para Java usando JUnit:

  Exportar o Teste:
        Clique em "File" > "Export" > "Export Test Suite As" e escolha o formato desejado, por exemplo, "Java JUnit".

    Exemplo de Teste Exportado para Java JUnit:

--> JAVA

package com.example;

import org.junit.After;
import org.junit.Before;
import org.junit.Test;
import org.openqa.selenium.By;
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;
import static org.junit.Assert.assertTrue;

public class GoogleSearchTest {
    private WebDriver driver;

    @Before
    public void setUp() {
        System.setProperty("webdriver.chrome.driver", "path/to/chromedriver");
        driver = new ChromeDriver();
    }

    @Test
    public void testGoogleSearch() {
        // Abrir Google
        driver.get("https://www.google.com");

        // Digitar "Selenium WebDriver" na barra de pesquisa
        driver.findElement(By.name("q")).sendKeys("Selenium WebDriver");

        // Submeter a busca
        driver.findElement(By.name("q")).submit();

        // Verificar se o resultado aparece
        boolean isResultPresent = driver.findElements(By.cssSelector("h3")).stream()
                .anyMatch(element -> element.getText().contains("Selenium WebDriver"));
        assertTrue(isResultPresent);
    }

    @After
    public void tearDown() {
        // Fechar o navegador
        if (driver != null) {
            driver.quit();
        }
    }
}

Notas Finais

1. Caminho do ChromeDriver: Certifique-se de que o caminho para o chromedriver está corretamente especificado no System.setProperty.
2. WebDriverWait: Em um código de produção, considere usar WebDriverWait para esperas explícitas e mais eficientes, ao invés de confiar em esperas implícitas.
3. Verificações: As verificações são essenciais para garantir que o comportamento esperado ocorra após as ações automatizadas.

Este tutorial fornece uma base sólida para começar com testes automatizados usando Selenium IDE, incluindo a entrada de dados e a verificação de resultados.
